# 🚀 Git 마이그레이션 설계 가이드
> ID: PRMT-GIT-MIGR-001 | Version: 2.1 | Last Updated: 2026-05-14

> [!IMPORTANT]
> 이 가이드는 [GLOBAL_RULES.md](./GLOBAL_RULES.md)를 최상위 지침으로 준수합니다. 파괴적 작업을 금지하며 설계 및 계획 수립에 집중합니다.

---

[전환 원칙]
- 모든 하위 폴더와 파일을 누락 없이 Git으로 이관하는 구조로 설계한다.
- 폴더 구조는 그대로 유지한다.
- 특정 파일이나 폴더를 임의로 제외하지 않는다.
- 오래된 소스도 제외하지 않는다.
- 전환 단계에서는 구조 변경, 리팩토링, 정리 작업을 수행하지 않는다.
- 전환 후 정리는 별도 단계에서 수행한다.

[분석 원칙]
- 파일 내용은 읽지 마십시오.
- 폴더 구조 중심으로 분석하십시오.
- 하위 폴더는 깊이 제한 없이 탐색하십시오.
- 최상위 폴더를 기본 Git 저장소 후보로 판단하십시오.
- src, lib, bin, obj, build, dist, node_modules 등은 중간/산출물 폴더로 간주하고 저장소 후보에서 제외하십시오.
- 확정 사실과 추정 사항을 반드시 구분하십시오.

[폴더 구조 출력 규칙]
- 파일명은 전체 나열하지 마십시오.
- 파일은 개수만 표시하십시오.
- 폴더 구조는 Git 전환 판단이 가능하도록 상세히 작성하십시오.
- 각 Git 저장소 후보별로 포함될 폴더 구조를 트리 형태로 작성하십시오.
- node_modules, bin, obj, build, dist, .git 등은 생략 가능하며 생략 사유를 명시하십시오.
- 너무 깊은 반복 구조는 ... 으로 축약하되, 생략된 구조를 설명하십시오.

[CSV 작성 규칙 - 매우 중요]
- 파일 인코딩은 UTF-8 with BOM으로 작성하십시오.
- Excel에서 한글이 깨지지 않도록 작성하십시오.
- 모든 필드는 큰따옴표(")로 감싸십시오.
- 구분자는 쉼표(,)를 사용하십시오.
- 줄바꿈이 포함된 값은 허용하지 않습니다.
- 한글 경로 및 설명이 깨지지 않도록 처리하십시오.

[CSV 컬럼 정의]
- "원본 폴더 경로"
- "권장 Git 저장소명"
- "저장소 유형" (단일/공통/보관/확인필요)
- "권장 이관 방식" (전체 import / 구조 유지 / SVN 구조 유지 등)
- "포함 범위" (전체 폴더 / 특정 루트 기준 등)
- "수동 업로드 여부" (예/아니오)
- "확인 필요 사항"
- "비고"

[수동 이관 절차 예시 - 참고 수준]

보고서에는 실제 실행 명령이 아닌 절차만 작성하십시오.

예:
- Git 저장소 수동 생성
- 로컬 폴더 기준 Git 초기화 준비
- 전체 파일 포함 여부 확인
- 최초 커밋 수행
- 원격 저장소 연결
- 수동 push
- Git 웹에서 파일 누락 여부 검증
