## 📋 PHASE 1: 계획 수립 (반드시 출력)

다음을 반드시 포함하라:

1. 현재 파일 상태 (존재/없음 가정)
2. 수정 대상 파일 + 변경 내용
3. 신규 생성 파일 목록
4. 전체 작업 순서 (의존성 고려)
5. 검증 방법 (실행 결과 기준)

※ 실제 파일 시스템 조회 없이 “표준 초기 상태” 기준으로 판단

## 📁 프로젝트 구조

아래 형식을 참고하여 작성하라. 명시된 구조 외 폴더 생성 금지.

```
[ProjectName]/
├── main.py               ← 진입점
├── requirements.txt      ← 의존성 목록
├── [module].py           ← 핵심 모듈 (기능별 분리)
├── config.py             ← 설정값 (하드코딩 금지)
└── tests/
    └── test_[module].py  ← 단위 테스트 (선택)
```

※ Web 서버: `app.py` + `routes/` + `models/` 추가 / CLI 도구: `cli.py` 추가 / GUI: `ui.py` 추가

---

## 🎯 기능 명세

아래 항목을 구체적으로 작성하라:

- **입력:** 어떤 데이터를 받는가 (파일 경로 / 사용자 입력 / API 요청 등)
- **처리:** 어떤 로직을 수행하는가 (알고리즘, 변환, 분석 등)
- **출력:** 어떤 결과를 내는가 (화면 출력 / 파일 저장 / HTTP 응답 등)
- **예외:** 어떤 오류 상황을 처리해야 하는가

---

## 🔧 구현 명세

### 필수 라이브러리
아래 형식으로 명시하라:
```
라이브러리명==버전  # 사용 목적
예: requests==2.31.0  # HTTP 클라이언트
```

---

### 핵심 로직
알고리즘 및 처리 흐름을 단계별로 명확히 작성하라:
```
1단계: 입력 수신 및 유효성 검사
2단계: [핵심 처리 로직]
3단계: 결과 포맷 변환 및 출력
```

---

### API / 주요 함수

각 엔드포인트 또는 핵심 함수를 아래 형식으로 명시하라:

```
함수명 / 엔드포인트:
  - 입력 (파라미터/타입):
  - 출력 (반환값/타입):
  - 처리 (주요 로직):
  - 예외 처리:
```

---

### 실행 방식

실행 명령을 정확히 명시하라:
```bash
# CLI 예시
python main.py --input [파일경로]

# Web 서버 예시
uvicorn app:app --host 0.0.0.0 --port 8000
```
