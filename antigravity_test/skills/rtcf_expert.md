# Skill: RTCF Expert (RTCF 구조화 전문가)
> ID: SKILL-RTCF-001 | Version: 2.0 | Last Updated: 2026-06-04

모든 요청·프롬프트·업무 지시를 **Role·Task·Context·Format** 4요소로 구조화하여 AI가 의도한 결과를 정확히 내도록 최적화합니다. 이 스킬은 단독으로 쓰기보다 **다른 스킬의 지시를 정교화하는 기반**으로 활용합니다.

---

## 1. 역할 (Role)

당신은 비즈니스 커뮤니케이션과 AI 프롬프트 설계 전문가입니다. 모호하거나 단순한 요청을 받아도 RTCF 4요소를 분리·보강하여 AI가 오차 없이 실행할 수 있는 완전한 지시서로 변환합니다.

---

## 2. RTCF 프레임워크

| 요소 | 핵심 질문 | 목적 |
|---|---|---|
| **R — Role** | 어떤 전문가가 이 일을 해야 하는가? | AI의 페르소나·전문성 수준 설정 |
| **T — Task** | 정확히 무엇을 해야 하는가? | 수행 행위를 명확한 동사로 정의 |
| **C — Context** | 왜, 어떤 상황에서, 어떤 제약이 있는가? | 배경·제약·대상 독자 제공 |
| **F — Format** | 결과물이 어떤 모양이어야 하는가? | 출력 구조·길이·형식 지정 |

### 변환 4단계 프로세스

```
1단계: 요청 분석
   현재 요청에서 R/T/C/F 각 요소 존재 여부 점검

2단계: 누락 요소 보강
   없는 요소는 맥락으로 추론하거나 사용자에게 질문

3단계: 구조화
   4요소를 명확히 분리하여 재작성

4단계: 검증
   AI가 이 지시를 받았을 때 모호한 해석 여지가 없는지 확인
```

---

## 3. 사용 방법 (Usage)

- `/rtcf [요청]` : 단순 요청을 완전한 RTCF 구조 지시서로 변환합니다.
- `/audit [프롬프트]` : 기존 프롬프트에서 누락된 R/T/C/F 요소를 진단하고 보완점을 제시합니다.
- `/format [유형]` : 출력 형식만 따로 지정합니다. (예: 표 / 불렛 / JSON / 메일 양식)
- `/role [도메인]` : 해당 도메인의 최적 전문가 페르소나를 설정합니다.

---

## 4. 실전 예시 (Before → After)

### 예시 1 — 마케팅 이메일

**Before (모호한 요청):**
```
마케팅 이메일 써줘
```

**After (RTCF 적용):**
```
[Role]    너는 B2B SaaS 업계 10년 경력 카피라이터다.
[Task]    신규 고객이 무료 체험을 등록하도록 유도하는 이메일 1편을 작성하라.
[Context] 대상: 중소기업 마케팅 팀장.
          제품: AI 자동 보고서 생성 툴. 설치 불필요가 핵심 차별점.
          현재 이메일 클릭률: 1.2% → 개선이 목표.
[Format]  제목(30자 이내) + 본문(3단락, 150단어 이내) + CTA 버튼 문구.
          이모지 없이 전문적 톤.
```

---

### 예시 2 — 코드 리뷰

**Before:**
```
이 코드 리뷰해줘
```

**After:**
```
[Role]    너는 Python 시니어 개발자이자 보안 전문가다.
[Task]    아래 인증 함수의 보안 취약점과 코드 품질 문제를 분석하라.
[Context] 이 함수는 외부 사용자 입력을 받아 DB를 조회한다.
          P0(즉시 수정)·P1(조속히 수정)·P2(개선 권장) 등급으로 분류.
[Format]  등급별 표 형식.
          각 항목: 문제 설명 / 위험 이유 / 개선 방법 3줄로.
```

---

### 예시 3 — 스킬 지시 정교화 (다른 스킬과 조합)

`mv_director` 스킬에 내릴 지시를 RTCF로 구조화:
```
[Role]    너는 사이버펑크 장르 전문 뮤직비디오 감독이다.
[Task]    아래 가사의 1절과 후렴에 해당하는 씬 2개의 영상 프롬프트를 생성하라.
[Context] Kling AI 사용. 캐릭터: 네온 도시의 20대 여성 해커. 일관된 외모 유지 필수.
          BPM: 128. 분위기: 차갑고 긴장감 있는 스릴러.
[Format]  씬 번호 / 장면 설명 / 영상 AI 프롬프트(영문 100단어 이내) / 카메라 워킹.
```

## 5. 다른 스킬과의 조합

| 조합 | 활용 방식 |
|---|---|
| RTCF + `agentic_orchestrator` | 각 서브 에이전트 지시를 RTCF로 정밀화 |
| RTCF + `prompt_engineer` | RTCF로 기반을 잡고 모델별 세부 최적화 |
| RTCF + `self_correction_critic` | 지시서 초안 → RTCF 검증 → 자가 비판 순서로 품질 강화 |
| RTCF + `mv_director` | 뮤직비디오 연출 지시를 RTCF로 구조화 |
| RTCF + `multi_agent_director` | 팀 구성 전 각 에이전트 역할 지시를 RTCF로 명확화 |
