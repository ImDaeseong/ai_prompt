# CLAUDE.md

> 이 문서는 모든 작업의 최상위 행동 지침입니다. GLOBAL_RULES.md를 대체하며, 개별 프롬프트 파일보다 우선합니다.

---

## 1. 언어 및 소통 원칙

- 모든 설명과 분석 결과는 **한글**로 작성한다.
- 기술 용어는 필요 시 한글·영문 병행 표기 가능.
- 전문적이고 간결하며, 명확한 근거를 바탕으로 답변한다.

## 2. 구현 전 사고 (Think Before Coding)

**가정하지 말고, 혼란을 숨기지 말고, 먼저 물어본다.**

- 요청이 모호하면 해석을 여러 개 제시하고 선택을 묻는다. 혼자 결정하지 않는다.
- 가정이 있다면 명시적으로 밝힌다.
- 더 단순한 접근이 있으면 먼저 말한다.
- 불분명한 게 있으면 구현 전에 멈추고 질문한다. 실수 후 질문은 비용이 크다.

## 3. 보안 및 안전 지침

- API Key, 비밀번호 등 민감 정보를 코드에 직접 포함하지 않는다.
- 명시적 허용 없이 외부 네트워크 호출 코드를 작성하지 않는다.
- 사용자 승인 없이 파일을 삭제·이동·덮어쓰는 코드를 생성하거나 실행하지 않는다.

## 4. 코드 및 산출물 품질

**요청한 것만, 완전하게, 기존 스타일 그대로.**

- **완전성:** 모든 코드는 실행 가능한 완전한 형태로 작성한다. 의사코드·TODO·생략 금지.
- **간결성:** 요청하지 않은 기능, 불필요한 추상화, 투기적 유연성을 추가하지 않는다. 200줄이 50줄로 줄 수 있으면 줄인다.
- **외과적 수정:** 요청 범위 밖의 코드는 건드리지 않는다. 인접 코드 개선·리팩터링·스타일 정리 금지. 변경된 모든 줄은 사용자 요청으로 직접 추적 가능해야 한다.
- **일관성:** 기존 프로젝트의 코드 스타일과 네이밍 컨벤션을 최우선으로 존중한다.
- **추측 금지:** 확실하지 않은 정보는 반드시 "확인 불가" 또는 "추정"임을 명시한다.

## 5. 목표 기반 실행 (Goal-Driven Execution)

**성공 기준을 먼저 정의하고, 충족될 때까지 루프한다.**

작업을 검증 가능한 목표로 변환한다:
- "버그 수정" → "버그를 재현하는 테스트를 작성하고, 통과시킨다"
- "기능 추가" → "해당 기능의 테스트를 먼저 작성하고, 통과시킨다"

여러 단계 작업이면 계획을 먼저 제시한다:
```
1. [단계] → 검증: [확인 방법]
2. [단계] → 검증: [확인 방법]
```

## 6. 토큰 비용 관리

- `CLAUDE.md`와 현재 작업에 필요한 프롬프트 1개만 사용한다.
- 이전 분석 결과나 대용량 문서는 요약·목차·핵심만 먼저 읽고, 필요 시 원문 확장.
- 사고 과정을 장황하게 출력하지 않는다. 근거와 결론 중심으로 요약한다.
- 빌드 산출물, 외부 라이브러리, 미디어, 로그, 바이너리는 분석 전 기본 제외.
- **컨텍스트 90% 도달 시**: `/compress` 명령으로 대화 컨텍스트를 압축하거나, HANDOFF.md에 이어받기 기록을 남기고 새 세션에서 계속한다.

## 7. UI 개발 필수 참조 스킬

UI 컴포넌트, 페이지, 앱 화면, 디자인 시스템 작업 시 아래 스킬을 **반드시 먼저 읽고 적용**한다.

- `antigravity_test/skills/frontend_design.md` — 웹 UI 디자인 철학, 타이포그래피, 컬러, 모션, 공간 구성
- `antigravity_test/skills/ui_ux_pro_max.md` — 전문 UI 규칙, Pre-Delivery 체크리스트, 접근성, 다크모드, 터치 타겟

적용 기준:
- 웹 컴포넌트/페이지/랜딩/대시보드 → `frontend_design.md` 우선
- 모바일 앱(Flutter, SwiftUI, React Native) → `ui_ux_pro_max.md` 우선
- 두 영역 모두 걸치면 두 파일 모두 참조

## 8. 문서화 규칙

- 표(Table), 리스트(Bullet), 코드 블록으로 가독성을 높인다.
- 분석 결과 파일 상단에는 분석 일시와 대상 범위를 명시한다.

## 9. 사용률 90% 이어받기 규칙 (Usage Handoff Rule)

- Claude Code 또는 Codex 사용률이 약 90%에 도달하면 작업을 더 진행하기 전에 이어받기 기록을 남긴다.
- 기록 항목: 현재 작업 목표, 완료된 항목, 미완료 항목, 관련 파일 경로, 실행한 명령, 검증 상태, 미해결 이슈, 다음에 바로 실행할 단계.
- 새 세션이 `HANDOFF.md`만 읽고 바로 이어서 진행할 수 있도록 `ai-workspace/HANDOFF.md`의 `Current State`를 업데이트한다.
- 이어받기 기록에는 비밀 정보, 인증 정보, 토큰, 내부 전용 식별자, 서버 주소, 고객 정보를 포함하지 않는다.
