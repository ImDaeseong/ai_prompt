=== [1단계] Visual Studio 버전 및 코드 스타일 분석 ===

[1-1. 빌드 환경 판별]

아래 기준으로 정확히 판단:

- .dsw / .dsp → VS6.0
- .vcproj → VS2008 이하
- .vcxproj → <PlatformToolset> 값 기준
  (v90=2008 / v100=2010 / v141=2017 / v142=2019 / v143=2022)

※ 여러 프로젝트 파일 존재 시:
- 주 빌드 환경
- 혼재 여부
를 구분해서 작성

---

[1-2. 코드 스타일 분석]

아래 항목 각각에 대해:
- 발견 여부 (O/X)
- 대표 파일명
- 코드 예시 (1줄)

분석 항목:
- NULL vs nullptr
- strcpy/sprintf vs *_s 계열
- #pragma once vs include guard
- 헝가리안 표기법 (m_, sz, n, b 등)
- CString vs std::string
- for문 변수 선언 위치
- 주석 스타일 (/* */ vs //)
- deprecated API 사용 여부

=== [3단계] 코드 품질 분석 ===

[분석 기준]

[P0 - 즉시 수정 필요 / 위험]
- 버퍼 오버런 (strcpy, sprintf 등)
- 메모리 누수 (new/delete 불일치)
- null 포인터 역참조
- 스택 오버플로우 위험

[P1 - 조속히 수정 필요]
- deprecated API 사용
- 64bit 호환성 문제
- MBCS/UNICODE 혼용
- 전역 변수 과다 사용

[P2 - 개선 권장]
- 코드 중복
- 매직 넘버
- 긴 함수 (>100줄)
- 주석 부족
