# Prompts

AI에 직접 복붙해서 사용하는 완성 프롬프트를 저장한다.

기술 패턴과 개발 교훈은 `dev/vibe-coding/skills/`에 기록한다.

## 아키텍처 / 설계

- [AI Architect 개발 흐름](ai-architect-workflow.md): Architect → Developer → Reviewer 흐름과 아키텍트 프롬프트 템플릿.
- [AI 생성 프로젝트 검증 설계](ai-project-validation.md): AI/Vibe Coding 결과물을 완료로 판단하기 위한 8단계 검증 기준.
- [LLM 서비스 설계](llm-service-architect.md): LLM API 기반 서비스의 모델 선택, 프롬프트 설계, 비용, 보안, 운영 흐름을 설계하는 Architect 프롬프트.
- [Multi-Agent System 설계](multi-agent-architect.md): 여러 AI 에이전트의 역할 분리, 오케스트레이션, 상태 관리, 비용, 보안을 설계하는 Architect 프롬프트.


## 품질 게이트

- [Project Lifecycle Quality Gate](project-lifecycle-quality-gate.md): 설계, 개발, 검증, 테스트 반복, 검수, 배포까지 전체 프로젝트 품질 흐름을 잡는 프롬프트.
- [Inspection First Quality Gate](inspection-first-quality-gate.md): 제품 개발보다 검수가 우선일 때 결과물의 신뢰성, 보안, 공개 가능성을 판정하는 프롬프트.

## Hermes 시작

- [Hermes Start](hermes-start.md): 세션 시작 시 Hermes 컨텍스트를 로드하는 기본 프롬프트.
