# AI 생성 프로젝트 검증 설계

AI나 바이브 코딩으로 만든 프로젝트는 빠르게 생성되지만, 설계 부족, 테스트 부족, 검증 부족 때문에 복구 불가능한 프로그램이 되기 쉽다. 그래서 완성 여부를 코드 생성량이나 파일 개수로 판단하지 않는다.

Hermes는 AI 생성 프로젝트를 완료로 판단할 때 목적, 설계, 실행 흐름, 테스트, 보안, 문서화, 유지보수 가능성, 제품화 가능성을 기준으로 본다.

## Core Principle

```text
생성됨 != 검증됨
동작함 != 제품화 가능함
파일이 많음 != 설계가 좋음
```

AI 생성 프로젝트는 아래 흐름을 통과해야 완료에 가까워진다.

```text
목적 확인
-> 설계 검토
-> 실행 검증
-> 테스트
-> 출력물 검수
-> 보안 점검
-> 문서화
-> 제품화 가능성 판단
```

## Validation Areas

### 1. 목적 검증

- 이 프로젝트가 해결하려는 문제가 한 문장으로 명확한가?
- 실제 사용자가 실행할 수 있는 흐름이 있는가?
- 기능 수가 아니라 실제 목표 달성에 도움이 되는가?

### 2. 설계 검증

- 폴더 구조, 데이터 흐름, API, 화면, 설정 파일의 역할이 분명한가?
- 핵심 로직과 입출력 경계가 분리되어 있는가?
- AI가 임시로 만든 중복 코드나 사용하지 않는 파일이 쌓이지 않았는가?
- 다음 수정자가 구조를 이해하고 복구할 수 있는가?

### 3. 실행 검증

- 설치 명령, 실행 명령, 기본 사용 흐름이 실제로 동작하는가?
- 주요 워크플로우가 처음부터 끝까지 이어지는가?
- Windows 경로, 한글 파일명, 인코딩, 공백 경로 같은 현실 조건을 고려했는가?

### 4. 테스트 검증

- 단위 테스트 또는 핵심 기능 테스트가 있는가?
- 빈 입력, 잘못된 파일, 누락된 설정, 중복 데이터, 실패 케이스를 다루는가?
- 테스트가 통과했다는 사실과 실행 명령이 문서에 기록되어 있는가?

### 5. 출력물 검수

- 생성된 파일이 실제로 필요한 형식과 내용을 갖췄는가?
- placeholder, 이전 프로젝트 이름, 이전 곡/고객/파일 잔여물이 없는가?
- 사람이 다시 판단할 수 있는 요약 또는 검토 문서가 있는가?

### 6. 보안 점검

- API Key, 토큰, 비밀번호, 회사 정보, 고객 정보, private URL이 없는가?
- 개인 설정 파일, IDE 설정, AI 도구 설정이 Git에 들어가지 않았는가?
- 공개 저장소로 올려도 되는 내용인지 확인했는가?

### 7. 문서 검증

- README에 사용법만이 아니라 현재 검증 상태가 적혀 있는가?
- 어떤 명령으로 테스트/검증했는지 남아 있는가?
- 알려진 한계와 다음 개선 방향이 기록되어 있는가?

### 8. 제품화 가능성

- 유지보수할 수 있는 구조인가?
- 반복 사용 가능한 도구인가, 일회성 산출물인가?
- 사용자, 비용, 배포, 운영, 지원 흐름을 생각했는가?
- 제품이나 서비스로 키울 가치가 있는 검증 결과가 있는가?

## Completion Rule

AI 생성 프로젝트는 아래 조건을 통과하기 전까지 완료로 보지 않는다.

- 목적이 명확하다.
- 핵심 사용 흐름이 실행된다.
- 테스트 또는 검증 명령이 있다.
- 실패 케이스가 최소한으로 검토되었다.
- 민감정보가 없다.
- README에 사용법과 검증 상태가 있다.
- 다음 작업자가 구조를 이해할 수 있다.

## Product Direction

이 검수/검증 체계는 장기적으로 `AI Project Quality Architect` 방향으로 확장될 수 있다.

가능한 형태:

- AI 프로젝트 검수 체크리스트
- 바이브 코딩 프로젝트 품질 진단 리포트
- GitHub 저장소 자동 분석 도구
- Claude/Codex/Roo용 검증 프롬프트 패키지
- AI 생성 코드 리뷰 Agent
- 출시 전 AI 앱 검수 서비스

처음부터 제품으로 만들지는 않는다. 먼저 개인 프로젝트에 적용하고, 반복되는 검증 항목을 체크리스트와 자동화 도구로 키운다.

## Hermes Judgment

AI 도구를 잘 쓰는 사람일수록 검증 설계가 필요하다. 빠르게 만드는 능력만으로는 충분하지 않다. Hermes는 AI 생성 프로젝트를 볼 때 항상 다음을 묻는다.

```text
만들었는가?
-> 검증했는가?
-> 유지보수 가능한가?
-> 보안상 안전한가?
-> 제품화 가능성이 있는가?
```
