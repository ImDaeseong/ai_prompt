# LLM 서비스 시장 전략 — AI 뮤직비디오 제작 관리 도구

이 문서는 LLM 서비스의 구조를 이해하고, 현재 본인 강점과 결합 가능한 서비스 방향을 정리한 전략 기준 문서다.

새 서비스를 설계하기 전에 이 문서를 먼저 읽는다.

---

## 1. LLM 서비스란 무엇인가

LLM 서비스 = LLM을 직접 만드는 것이 아니라, LLM을 이용해서 문제를 해결하는 서비스를 만드는 것.

```
사용자
 ↓
자사 서비스 (당신이 만드는 부분)
 ↓
GPT / Claude / Gemini API
 ↓
결과 가공
 ↓
사용자
```

대부분의 스타트업, 한국 AI 서비스 회사는 GPT, Claude, Gemini API를 사용한다.
직접 LLM을 학습시키는 회사는 극소수다.

핵심 질문은 "어떤 LLM을 쓰느냐"가 아니라 **"LLM으로 어떤 문제를 해결하느냐"**다.

## 2. 시장 경쟁 구조 (3단계)

### 1단계 — 단순 프롬프트 생성기 (경쟁 많음, 가치 낮음)

```
주제 입력 → GPT → 프롬프트 생성
```

문제: 사용자가 ChatGPT에 직접 물어봐도 비슷한 결과.
차별성 없음. 수익화 어려움.

### 2단계 — 반자동 제작 서비스 (경쟁 많음, 품질 차이 존재)

```
주제 → 가사 생성 → 이미지 프롬프트 → 영상 프롬프트 → 편집 가이드
```

이미 경쟁자 많음. 하지만 품질 차이가 크다.
대부분 캐릭터 일관성이 없어서 매 장면 인물이 바뀜.

### 3단계 — 캐릭터 일관성 + 제작 워크플로우 관리 (경쟁 적음, 가치 높음)

```
프로젝트 생성
 ↓
캐릭터/세계관 고정
 ↓
장면별 프롬프트 자동 생성 (캐릭터 일관성 유지)
 ↓
컷 구성 관리
 ↓
영상 관리 + 편집표 생성
 ↓
유튜브 최적화 (제목, 설명, 태그)
```

현재 시장에서 가장 부족한 영역. 여기서 가치가 생긴다.

---

## 3. 현재 시장에서 가장 부족한 것

부족한 것:
- 노래 생성 X (이미 많음)
- 이미지 생성 X (이미 많음)
- 영상 생성 X (이미 많음)

실제로 부족한 것:
- 여러 곡의 제작 상태 관리
- 어떤 컷을 만들었는지 추적
- 캐릭터 일관성 유지 (같은 밴드, 같은 세계관)
- 영상 제작 워크플로우 전체 관리

---

## 4. 본인 강점 조합

| 강점 | 활용 방향 |
|---|---|
| Android 개발 경험 | Android 개발자용 AI 도구, 모바일 UI |
| Suno 음악 제작 경험 | 실제 사용자 관점의 제작 흐름 이해 |
| ai_img_video_prompt 개발 경험 | 203곡 검증, 캐릭터 일관성 프롬프트 설계 |
| 사이버펑크 해골 밴드 세계관 | 고정 세계관 + 캐릭터 일관성의 실증 사례 |
| GPT/Claude API 경험 | 다중 LLM 조합 가능 |

## 5. 서비스 방향

### 핵심 정의

**AI 뮤직비디오 제작 프로젝트 관리 도구**

단순 프롬프트 생성기가 아니라, 제작 전체 흐름을 관리하는 도구.

```
사용자 입력 (곡 정보, 캐릭터, 세계관)
 ↓
GPT / Claude
 ↓
가사 분석
장면 구성
이미지 프롬프트 생성 (캐릭터 일관성 유지)
영상 프롬프트 생성 (Kling)
편집표 생성 (CapCut)
유튜브 패키지 생성 (제목, 설명, 태그)
 ↓
프로젝트별 버전 관리 + 검수 문서
```

### 경쟁력 포인트

GPT 호출 자체가 아니라 다음에서 나온다:

- 캐릭터 고정 프롬프트 설계 품질
- 장면 구성 자동화 수준 (30컷 이상)
- 프로젝트 간 일관성 유지 방법
- 사람이 검수할 수 있는 요약 문서 생성
- 실제 Suno + Kling + CapCut 워크플로우와 연결

## 7. Architect 프롬프트

이 서비스를 실제로 설계할 때 아래 프롬프트를 사용한다.

[multi-agent-implementation-guide.md](./multi-agent-implementation-guide.md)를 함께 참고한다.

```text
너는 20년 경력의 AI 서비스 아키텍트다.

코드를 작성하지 말고 설계만 수행하라.

아래 서비스를 설계하라.

서비스: AI 뮤직비디오 제작 프로젝트 관리 도구
사용 LLM: GPT-4o / Claude 3.5 Sonnet (조합 가능)
핵심 기능:
  - 곡 정보 + 캐릭터 + 세계관 입력
  - 장면 구성 자동 생성 (30컷 기준)
  - 이미지 프롬프트 생성 (캐릭터 일관성 유지)
  - 영상 프롬프트 생성 (Kling 최적화)
  - 편집표 생성 (CapCut 호환)
  - 유튜브 패키지 생성 (제목, 설명, 태그)
  - 프로젝트별 버전 관리 및 검수 문서 생성
대상 사용자: AI 뮤직비디오 제작자, 유튜브 AI 음악 채널 운영자
배포 환경: Windows 로컬 앱 또는 웹

다음 순서로 설계하라:

1. 서비스 목적 한 줄 정의
2. 핵심 차별성 (경쟁자와 다른 점)
3. 시스템 구조 (데이터 흐름)
4. LLM 역할 분리 (어떤 LLM이 어떤 역할)
5. 캐릭터 일관성 유지 설계
6. 프롬프트 버전 관리 방법
7. 비용 설계 (API 호출 비용 최적화)
8. 개발 Task 분해 (1개부터 시작 가능하도록)
9. 위험 요소
10. 무료/저비용으로 먼저 검증할 방법
```

## Hermes Judgment

이 문서는 분석이다. 분석이 끝나면 실행으로 돌아간다.

```text
지금 실행 중인 것:
E:\ai_test\ai_img_video_prompt — 이미 이 방향의 시작점

다음 판단 기준:
현재 채널 프로그램이 완성되면 → 이 문서를 꺼내서 Architect 프롬프트 사용
지금 당장 새 서비스 설계로 이탈하지 않는다
```

이 문서는 `ai_prompt` 저장소로 이동한다. Hermes 코어 파일이 아니므로 ai_prompt에 저장 가능하다.
