## 무한 루프와 토큰 낭비가 생기는 근본 원인

> Simon Willison: "LLM은 완료 기준이 없으면 '더 나은 방법'을 영원히 시도한다."

```
원인 1: 태스크 범위가 불명확하다 → AI가 임의로 범위를 확장한다
원인 2: 성공 기준이 없다 → AI가 언제 멈춰야 할지 모른다
원인 3: 컨텍스트가 너무 길어졌다 → AI가 초기 목적을 잃는다
원인 4: 한 세션에 너무 많은 것을 요청한다 → 복합 오류가 서로 엉킨다
원인 5: 실패 후 같은 방식으로 재시도한다 → 농약 역설 발생
```

## 섹션 1: 세션 시작 전 필수 준비

> Geoffrey Huntley: "AI 세션 전 git commit은 안전벨트다. 선택이 아니다."
> Martin Fowler: "AI에게 코드를 맡기기 전, 지금 상태를 반드시 저장하라."

### 세션 시작 체크리스트

- [ ] **git commit** — 현재 상태를 커밋한다 (AI 작업 실패 시 롤백 기준점)
- [ ] **브랜치 생성** — 실험적 변경은 별도 브랜치에서 한다
- [ ] **CLAUDE.md 확인** — 프로젝트 컨텍스트가 최신 상태인지 확인
- [ ] **세션 목적 한 문장** — 지금 AI에게 시킬 일을 한 문장으로 정의한다
- [ ] **완료 기준 정의** — "이렇게 되면 이 세션은 끝난다"를 미리 정한다
- [ ] **범위 제한** — 이번 세션에서 수정할 파일 목록을 미리 정한다

```
세션 목적:
완료 기준:
수정 허용 파일:
  - 파일명 1
  - 파일명 2
수정 금지 파일:
  - (예: 결제 모듈, 인증 모듈)
예상 소요 시간:
최대 재시도 횟수: 3회
```

## 섹션 2: CLAUDE.md 설정 (Claude Code 공식 문서 기반)

> Claude Code 공식 문서: "CLAUDE.md는 모든 세션이 시작될 때 자동으로 읽힌다.
> 프로젝트 컨텍스트, 금지 사항, 코딩 규칙을 여기에 넣으면 매번 설명하지 않아도 된다."

### 프로젝트 CLAUDE.md 최소 구성

```markdown
# CLAUDE

## 프로젝트 개요
[한 문장 설명]

## 기술 스택
[언어, 프레임워크, 주요 라이브러리]

## 절대 수정 금지 파일
- [파일명]: [이유]

## 절대 하지 말아야 할 것
- API 키, 비밀번호를 코드에 직접 쓰지 않는다
- .env 파일을 생성하거나 수정하지 않는다
- 테스트 없이 핵심 로직을 변경하지 않는다
- 한 번에 3개 이상의 파일을 동시에 수정하지 않는다

## 코딩 컨벤션
[네이밍, 들여쓰기, 주석 방식 등]

## 완료 기준
[이 프로젝트에서 "완료"의 기준]

## 환경
[Node 버전, Python 버전 등]
```

### CLAUDE.md 위치 우선순위 (공식 문서 기준)

```
1순위: ./CLAUDE.md (프로젝트 루트) — 프로젝트 전용 규칙
2순위: ~/.claude/CLAUDE.md (홈 디렉토리) — 모든 프로젝트 공통 규칙
```

**공통 규칙 추천 항목 (~/.claude/CLAUDE.md):**
```markdown
# 전역 규칙

## 항상 적용
- API 키, 토큰, 비밀번호를 코드에 직접 쓰지 않는다
- 한 세션에 하나의 태스크만 처리한다
- 같은 방법으로 3회 실패하면 중단하고 보고한다
- 수정 전 현재 파일 상태를 먼저 이해했는지 확인한다
```

## 섹션 3: 효과적인 프롬프트 작성법 (CRAFT 원칙)

> Simon Willison: "AI 프롬프트의 품질이 결과의 품질을 결정한다."

```
C — Context  : 지금 무엇을 하고 있는가 (배경)
R — Request  : 정확히 무엇을 해달라는가 (요청)
A — Approach : 어떤 방식으로 해달라는가 (제약)
F — Format   : 결과를 어떤 형태로 달라는가 (출력 형식)
T — Terminal : 언제 멈춰야 하는가 (완료 기준)
```

**나쁜 프롬프트:** `"이 함수가 버그가 있어, 고쳐줘"` → 범위 없음, 무한 수정
**좋은 프롬프트:**
```
[Context]  auth.js의 login()에서 JWT 토큰 만료 시
[Request]  401 대신 500이 반환되는 버그 수정
[Approach] login() 함수만 수정. 다른 파일 건드리지 마
[Format]   수정된 코드만. 설명은 한 줄로
[Terminal] 만료 토큰 → 401 반환이 되면 완료
```

### 자주 쓰는 패턴

**버그 수정:**
```
[파일명]의 [함수명]에서 [증상]이 발생한다.
[해당 함수]만 수정해줘.
수정 후 [재현 조건]에서 [기대 결과]가 나오면 완료.
3회 시도 후 해결 안 되면 원인 분석만 보고해줘.
```

**신규 기능:**
```
[파일명]에 [기능] 함수를 추가해줘.
조건: [제약], 입력: [파라미터], 출력: [반환값]
다른 기존 함수는 수정하지 마.
```

## 섹션 4: 무한 루프 감지 및 탈출

> Kent Beck: "같은 방법으로 3번 실패했다면, 방법이 틀린 것이다."

### 무한 루프 신호 감지 — 하나라도 해당하면 즉시 중단

| 신호 | 판단 | 행동 |
|---|---|---|
| 같은 에러 3회 이상 반복 | 구조적 문제 | 세션 중단, 재설계 |
| AI가 "다른 방법 시도" 2회 이상 | 방향 상실 | `/clear` 후 새 접근 |
| 수정 파일이 처음 범위 벗어남 | 범위 이탈 | 즉시 중단, git restore |
| 코드가 수정할수록 길어짐 | 과잉 생성 | 중단, 직접 단순화 |
| AI 응답이 이전 응답과 모순 | 컨텍스트 오염 | `/clear` 필수 |
| 토큰 사용량 급증 | 컨텍스트 초과 | 세션 종료 후 재시작 |

### 탈출 절차

```
Step 1: AI 응답 중단 (Escape 또는 Stop)
Step 2: git diff로 현재 변경 내용 확인
Step 3: 범위 이내인가?
        → Yes: 수동 검토 후 커밋
        → No:  git restore . (롤백)
Step 4: 원인 분류 → 컨텍스트 오염: /clear / 구조적: 재설계 / 프롬프트: 재작성
```

### 컨텍스트 관리 (Claude Code 공식 문서)

```bash
/clear                    # 컨텍스트 초기화 — 태스크 전환 시 반드시
claude --no-history       # 히스토리 없이 새 세션
@파일명.ts 함수명만 봐줘   # 필요한 파일만 명시적 참조
claude --continue         # 이전 세션 이어가기 (의도적일 때만)
```

**초기화 타이밍:** 새 태스크 전환 / 3회 이상 실패 / 대화 10회 초과 / 목적에서 멀어졌을 때

## 섹션 5: 토큰 효율화

| 낭비 패턴 | 해결 |
|---|---|
| 전체 파일 붙여넣기 | `@파일명` 참조만 |
| "전체 코드 다시 써줘" | "이 함수만 수정" |
| 긴 설명 요청 | "3줄로" / "핵심만" |
| 대화가 길어진 채 진행 | `/clear` 후 핵심만 재입력 |

**30분 타임박스 (Kent Beck):** 30분 내 해결 안 되면 → 태스크 분리 또는 접근법 변경

## 섹션 6: VS Code 워크스페이스 설정

### 권장 .vscode/settings.json

```json
{
  "editor.inlineSuggest.enabled": true,
  "github.copilot.enable": { "*": true, "plaintext": false, "markdown": false },
  "files.exclude": { "**/.env": true, "**/secrets": true },
  "editor.formatOnSave": true,
  "git.confirmSync": true
}
```

### .gitignore AI 관련 필수 항목

```gitignore
.claude/
*.claude-session
.env
.env.local
secrets/
*.key
*.pem
```

### GitHub Copilot Instructions 파일

```markdown
# .github/copilot-instructions.md
- API 키, 비밀번호를 코드에 포함하지 않는다
- 한 번에 한 함수 단위로 수정을 제안한다
- 기존 코드 스타일을 유지한다
```

## 섹션 7: Claude Code vs Copilot/Codex 비교

### Claude Code

| 항목 | 내용 | 팁 |
|---|---|---|
| 컨텍스트 파일 | CLAUDE.md 자동 로드 | 프로젝트마다 CLAUDE.md 필수 |
| 파일 참조 | `@파일명` | 필요한 파일만 명시 |
| 세션 초기화 | `/clear` | 태스크 전환 시 반드시 |
| 금지 | `--dangerously-skip-permissions` | **절대 사용 금지** |

### Copilot / Codex

| 항목 | 내용 | 팁 |
|---|---|---|
| Inline Chat | 코드 선택 후 `Ctrl+I` | 가장 정확한 범위 제한 |
| 컨텍스트 | 열린 파일들 | 관련 없는 탭 닫기 |
| `#file` 참조 | 특정 파일 지정 | `#파일명`으로 범위 명확화 |
