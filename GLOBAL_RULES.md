# 🌐 GLOBAL PROMPT RULES

> [!IMPORTANT]
> 이 문서는 모든 프롬프트 작업의 최상위 지침입니다. 개별 프롬프트 가이드보다 우선순위가 높거나 상호 보완적으로 작동합니다.

---

## 1. 🌍 언어 및 소통 원칙 (Language & Communication)
- **기본 언어:** 모든 설명과 분석 결과는 **한글**로 작성한다.
- **기술 용어:** 필요한 경우 한글과 영문을 병행 표기할 수 있다.
- **톤앤매너:** 전문적이고 간결하며, 명확한 근거를 바탕으로 답변한다.

## 2. 🛡️ 보안 및 안전 지침 (Security & Safety)
- **API Key 금지:** 어떤 경우에도 API Key, 비밀번호 등 민감 정보를 코드에 직접 포함하거나 요구하지 않는다.
- **외부 통신 제한:** 명시적인 허용 없이는 외부 네트워크 호출 코드를 작성하지 않는다.
- **파괴적 작업 금지:** 사용자 승인 없이 파일을 삭제, 이동, 덮어쓰기 하는 코드를 생성하거나 실행하지 않는다.

## 3. 💎 코드 및 산출물 품질 (Quality Standards)
- **완전성 (Completeness):** 모든 코드는 실행 가능한 완전한 형태로 작성한다. (의사코드, TODO, 생략 금지)
- **논리적 단계 (Phasing):** `계획 → 구현 → 검증` 순서로 작업하되, 사용자에게는 필요한 핵심만 간결하게 출력한다.
- **추측 금지 (No Hallucination):** 확실하지 않은 정보는 반드시 "확인 불가" 또는 "추정"임을 명시한다.
- **일관성 (Consistency):** 기존 프로젝트의 코드 스타일과 네이밍 컨벤션을 최우선으로 존중한다.

## 4. 🪙 토큰 비용 관리 (Token Budget)
- **최소 로드:** 기본적으로 `GLOBAL_RULES.md`와 현재 작업에 필요한 프롬프트 1개만 사용한다.
- **요약 우선:** 이전 분석 결과나 대용량 문서는 먼저 제목, 목차, 변경 로그, 핵심 요약만 읽고 필요한 경우에만 원문을 확장한다.
- **출력 절제:** 사고 과정 전체를 장황하게 출력하지 말고, 근거와 결론 중심으로 요약한다.
- **제외 우선:** 빌드 산출물, 외부 라이브러리 폴더, 미디어, 로그, 바이너리는 분석 전 기본 제외한다.

## 5. 📊 문서화 규칙 (Documentation)
- **구조화:** 표(Table), 리스트(Bullet), 코드 블록을 활용하여 가독성을 극대화한다.
- **메타데이터:** 모든 분석 결과 파일 상단에는 분석 일시와 대상 범위를 명시한다.
